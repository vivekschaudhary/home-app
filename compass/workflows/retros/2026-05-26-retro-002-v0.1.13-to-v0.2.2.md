# Retro #002 — improvements v0.1.13 to v0.2.2

> Second retro, backfilled retroactively as part of v0.2.8 shipping. Covers Compass's major capability expansion phase. **Status: archive.**

## Improvements in scope

- **v0.1.13** — `/create-bet-portfolio` (MVP wedge bootstrap; stub-promotion via `/create-brief`)
- **v0.1.14** — `/plan` workflow (living time-bound schedule; auto-trigger from `/advance`)
- **v0.2.0** — `/scan` continuous quality scanner (Snyk-style; 44 checks across 6 SDLC phases; Scanner role; suppression policy) — **MAJOR**
- **v0.2.1** — `/dashboard` single-file HTML view (auto-trigger from `/scan`, `/plan`, `/metrics`, `/status`)
- **v0.2.2** — Gitignore `docs/dashboard.html` (derived view convention)

## Common patterns (positive)

| Pattern | Instances in this batch | What it means |
|---|---|---|
| **New-capability releases** | v0.1.13, v0.1.14, v0.2.0, v0.2.1 — **4 of 5 improvements** | Compass's surface area roughly doubled in this batch. New workflows, new role (Scanner), new HTML artifact. Expansion phase. |
| **`status: living` for derived artifacts** | `/plan` (v0.1.14), `/scan` reports (v0.2.0), `/dashboard` (v0.2.1) — **3 instances** | Convention-ready. Compass needed a status value for "never finalized" artifacts. (Codified informally; AGENTS.md should make this explicit.) |
| **State-detection table at workflow top** | `/create-bet-portfolio`, `/plan`, `/scan` — **3 instances** | Convention-ready. New workflows started shipping with this shape as default. |
| **Auto-trigger chain** | `/advance` → `/plan` → `/scan` → `/dashboard` built incrementally across v0.1.14, v0.2.0, v0.2.1 | Visibility never goes stale during normal workflow usage. Load-bearing for the framework's "rolling visibility" promise. |

## Recurring anti-patterns (negative)

| Anti-pattern | Instances | Hardening shape applied | Convention-ready? |
|---|---|---|---|
| **Rubric framing → engineer tune-out** | v0.2.0's reframing from "rubric self-assessment" to "Snyk-style findings" | Mental-model swap: findings + severity + confidence + suppression replaced "did I do X? Y/N." Same constraint, different framing engineers actually trust. | **Yes — one instance but huge implication.** Naming this as an anti-pattern would help future framework design avoid rubric-shaped enforcement when scanner-shaped enforcement is available. |
| **Derived-view commit churn** | v0.2.2 caught `docs/dashboard.html` regenerating on every workflow run; ~2500 lines of diff per commit | `.gitignore` rule + explicit "gitignore only pure views with no user-relevant state" guidance | First instance; will likely recur if more derived views ship. |

## Convention candidates

| Name | Instances | Proposed principle text | Recommendation |
|---|---|---|---|
| **`status: living` for derived artifacts** | 3 in this batch (+ playbooks later — 4 by v0.2.7) | "Derived artifacts that are regenerated continuously use `status: living`, never `proposed` / `approved`. Hand-editing them is anti-pattern; the source artifacts they read are the canonical state." | **Promote.** 3 instances + clear semantics. |
| **State-detection table at workflow top** | 3 in this batch (+ /scan in v0.2.0) | "Workflows that gate based on artifact state begin with a State Detection table — explicit (state, action) pairs covering missing-artifact / proposed / approved / done. Refuse explicitly for ambiguous states." | **Promote.** Convention is clear and reduces reasoning load. |
| **Auto-trigger chain** | Established this batch | "Workflows that produce derived state auto-invoke downstream derived workflows at their end. /advance triggers /plan → /scan → /dashboard transitively. New derived workflows should slot into this chain." | **Document the chain but watch usage.** (See drift signals.) |
