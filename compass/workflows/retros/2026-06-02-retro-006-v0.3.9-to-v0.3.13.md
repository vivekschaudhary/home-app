- **Will Retro #007 fire on time after improvement #35?** Hard line still in effect from Retro #005. 3rd consecutive on-time retro would be very strong validation.
- **Will `[hard-line-declaration]` generalize to non-cadence commitments?** All 3 instances are cadence/sequence commitments. A 4th instance with a different shape (e.g., feature commitment, scope commitment, quality commitment) would validate generalizability.
- **Will the v0.3 → v0.4 transition need a milestone marker?** v0.3.13 is the first user-facing v0.4-shaped deliverable. At some point the framework jumps from v0.3.x to v0.4.x — what's the criterion? "Delivery Manager runtime exists" is the obvious answer; worth deciding criterion explicitly.
