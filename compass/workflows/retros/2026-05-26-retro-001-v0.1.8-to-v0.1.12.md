# Retro #001 — improvements v0.1.8 to v0.1.12

> First retro, backfilled retroactively as part of v0.2.8 (when `/retro` workflow shipped). Covers Compass's first 5 substantive framework improvements. Per AGENTS.md principle #14 — soft-spec-rationalization defense via periodic pattern review. **Status: archive.**

## Improvements in scope

- **v0.1.8** — Researcher 6-category source guide + Defensibility (9-moat) for foundational product bets
- **v0.1.9** — Researcher must produce evidence; ban "log-and-walk-away" as a deliverable
- **v0.1.10** — Architecture rename completion (`setup-architecture` → `setup-foundation-architecture`; `create-architecture` → `create-bet-architecture`); improvements log consolidation
- **v0.1.11** — Architect 6 Well-Architected pillars + 6 architecture-research categories + Phase A/Phase B split with HITL hard gate before scaffold + ADR via versioning
- **v0.1.12** — Foundational data model derived before DB choice (with Mermaid ERD)

## Common patterns (positive)

| Pattern | Instances in this batch | What it means |
|---|---|---|
| **N-category framework with cite-or-mark-n/a enforcement** | v0.1.8 (Researcher 6) · v0.1.11 (Architect 6-pillar AND 6-research) — **3 instances** | Shape stabilizing. Already convention-ready by improvement #5 — though we didn't formally name it until much later. |
| **Decide-before-derive ordering** | v0.1.11 (fitness functions before stack) · v0.1.12 (data model before DB) — **2 instances** | Architecture decisions cascade; foundation-level decisions must precede dependent choices. |
| **Refuse + escalate to upstream artifact** | v0.1.9 (Researcher refuses gap-only deliverable) · v0.1.11 (Architect refuses scaffold pre-HITL) · v0.1.12 (DB choice refused until data model) — **3 instances** | Same convention-ready shape as N-category. |

## Recurring anti-patterns (negative)

| Anti-pattern | Instances | Hardening shape applied | Convention-ready? |
|---|---|---|---|
| **Soft spec verb → AI rationalized shortcut** | v0.1.9 ("engage" → "engaged = logged the gap"); v0.1.11 ("smart defaults" → "I'll pick whatever") | Explicit imperative ("do NOT log without evidence" / "score against 6 pillars; empty cells fail") + verification gate + named anti-pattern ("log-and-walk-away" / "smart defaults") | **Yes — 2 instances of the exact same recipe**. This will recur many more times. |

## Convention candidates

| Name | Instances | Proposed principle text | Recommendation |
|---|---|---|---|
| **N-category cite-or-mark-n/a enforcement** | 3 in this batch | "Structured consultation frameworks use N named categories; each produces a citation OR explicit `n/a — <reason>` note; empty cells fail." | **Promote now** (3 instances, mature shape). NOTE: actually promoted in v0.2.8 — 17 improvements later than this batch. **5-cycle convention-discovery lag.** |
| **Refuse + escalate to upstream artifact** | 3 in this batch | "When a workflow detects an attempt to silently widen the scope of an upstream decision, refuse and escalate to the workflow that owns that decision." | **Promote now** (3 instances, mature shape). Same 17-improvement lag as above. |
| **Soft-spec hardening recipe** (constraint + gate + named anti-pattern) | 2 in this batch | "Every load-bearing constraint requires explicit imperative language + mechanical verification gate + named anti-pattern. Specs that depend on agent judgment will eventually be wrong." | **Promote now** even on 2 instances — this is the foundational pattern. (Actually promoted in v0.2.8 as principle #14.) |
